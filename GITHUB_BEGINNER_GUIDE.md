# GitHub 初心者向けガイド - WoW プロジェクト

## 📋 はじめに

このガイドは、GitHub を使用したチーム開発の基本的な流れを説明しています。
毎日これに従って操作すれば、問題なく開発できます。

---

## 🚀 毎日の開発フロー

### **朝（開発開始時）**

```
1. PowerShell を開く
2. プロジェクトフォルダに移動
3. 最新コードをダウンロード
```

**実行するコマンド：**
```powershell
cd "C:\Users\obaka\OneDrive\Desktop\気温分割完了 0412 - ログイン画面付き\気温分割完了 0412"
git pull origin main
```

**出力例：**
```
Already up to date.
（または新しいファイルがダウンロードされる）
```

---

### **昼～夜（開発中）**

```
1. コードを編集
2. ファイルを保存
3. ブラウザで動作確認
```

**特に難しいことはありません。いつもどおり開発してください。**

---

### **夜（開発終了時）**

```
1. 自分の編集内容を確認
2. GitHub にアップロード（push）
```

**実行するコマンド：**

```powershell
# Step 1: 編集したファイルの確認
git status

# Step 2: ファイルをまとめる
git add .

# Step 3: 説明付きで保存
git commit -m "ログイン機能を実装しました"

# Step 4: GitHub にアップロード
git push origin main
```

---

## 🎯 各コマンドの意味

| コマンド | 何をするのか | いつ実行 |
|---------|-----------|--------|
| `git pull origin main` | GitHub から最新コードをダウンロード | 朝（開発前） |
| `git status` | 今の状態を確認（どのファイルが変わったか） | 任意 |
| `git add .` | 編集したファイルを「提出準備」 | 夜（開発後） |
| `git commit -m "説明"` | 編集内容を説明付きで保存 | 夜（開発後） |
| `git push origin main` | GitHub にアップロード | 夜（開発後） |

## 📍 GitHub 上でできること

### **1. コードを確認する**
- GitHub.com → WoW リポジトリ → Code タブ
- ファイル一覧が表示される
- ファイルをクリックすると、内容が見られる

### **2. コミット履歴を確認する**
- GitHub.com → WoW リポジトリ → Code タブ
- ページをスクロールダウン
- 「Commits」セクションに誰が何をいつ編集したかが表示される

### **3. Issues で質問・相談する**
- GitHub.com → WoW リポジトリ → Issues タブ
- 「New Issue」ボタンをクリック
- チームメンバーに質問・相談できる

### **4. Pull Request で編集内容を提案する（中級）**
- 後日説明予定

## ⚠️ 重要な注意点

### **1. .env.local は絶対に push しない！**

`.env.local` には秘密情報（Supabase キー）が含まれています。

**これが含まれていたら危険です：**
```
❌ git push を実行前に必ず確認

$ git status
...
.env.local  ← これが見えたら危険！
```

**安全：** `.gitignore` に `.env.local` が含まれているので、通常は自動的に除外されます。

### **2. コンフリクト（衝突）について**

複数人が同じファイルを編集すると、衝突が起こることがあります。

**もし起こったら：**
```
1. PowerShell に「conflict」というメッセージが表示される
2. チームリーダー（あなた）に相談する
3. ファイルを手動で修正する
```

## 🆘 よくあるエラーと対処法

### **エラー1: 「fatal: not a git repository」**

```
原因：正しいフォルダにいない
解決：cd コマンドで正しいフォルダに移動する

cd "C:\Users\obaka\OneDrive\Desktop\気温分割完了 0412 - ログイン画面付き\気温分割完了 0412"
```

### **エラー2: 「error: failed to push some refs」**

```
原因：他の人の編集と衝突している
解決：まず git pull を実行してから git push

git pull origin main
git push origin main
```

### **エラー3: 「Author identity unknown」**

```
原因：Git の設定がない
解決：以下を実行（最初の1回のみ）

git config --global user.email "あなたのメール"
git config --global user.name "あなたの名前"
```

## 📞 チーム開発での流れ

### **シナリオ：3人でログイン機能を実装する場合**

```
朝:
 Aさん → git pull → 最新版を取得
 Bさん → git pull → 最新版を取得
 Cさん → git pull → 最新版を取得

昼:
 Aさん → ログイン画面を編集 → git push
 Bさん → ログイン機能のロジックを編集 → git push
 Cさん → テストコードを追加 → git push

夕方:
 Aさん → git pull → Bさんとcさんの編集を反映
 Bさん → git pull → Aさんとcさんの編集を反映
 Cさん → git pull → Aさんとbさんの編集を反映

完成:
 全員が最新版で動作確認
```

## ✅ チェックリスト

開発を始める前に、毎日確認してください：

- [ ] PowerShell を開いた
- [ ] 正しいフォルダにいることを確認した（`pwd` コマンドで確認）
- [ ] `git pull origin main` を実行した
- [ ] ブラウザで最新版を確認した
- [ ] 開発を開始した

開発を終える前に、毎日確認してください：

- [ ] コードを保存した（Ctrl+S）
- [ ] `git status` で変更ファイルを確認した
- [ ] 他の人の編集と衝突していないか確認した
- [ ] `git commit -m "説明"` で保存した
- [ ] `git push origin main` で GitHub にアップロードした
- [ ] GitHub.com で自分の編集が反映されているか確認した
