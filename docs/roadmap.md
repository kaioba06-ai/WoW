## 完成形仕様

タブ: ユーザーIDシート + DebugLog

| 行 | 役割 | 内容 |
|---|---|---|
| 1-2 | プロフィール 22項目 | 性別・年代・身長・体重・体格・骨格・肩幅・胸囲・首回り・裄丈・腹囲・ウエスト・ヒップ・股下・太もも・靴・手首・肌の色・顔の形・髪型・髪色 |
| 3-4 | アバター生成プロンプト（英語） | A3ラベル / 行4プロンプト本文 |

**※ U列廃止**: 旧コーデ画像列は廃止し、シーン画像（C14-17）に統合。

### UI（home.html）

```
┌─ 天気バー ─────────────────────────────────────┐
│ 天気アイコン MAX/MIN  地名                      │
├──────────────────────────────┬─────────────────┤
│  【メインカード】             │  スロット① NOW │ ← C14
│   C列シーン画像（選択中）     │  スロット② +3h│ ← C15
│                              │  スロット③ +6h│ ← C16
│  【ルックバナー】             │  スロット④+12h│ ← C17
│   コーデ名(V) ワンポイント(W) │                 │
│   気温(B) [🔄][👆]            │                 │
└──────────────────────────────┴─────────────────┘
```

- サムネタップでメイン＋バナーが切替、初期=NOW
- 🔄ボタンで `regenerate_outfits=true`
- 👆ボタンはダミー配置（仕様未定・保留）
- **モバイル最優先**、PCは副次

### パイプライン（5回呼び出し/再生成）

1. **コーデJSON生成** (1回)
   - 入力: プロフィール + 4スロット予報 + 感度
   - 出力: `{slots:[{outfit_name, one_point(JA), parts:{head,...,accessory}}]×4}`
2-5. **シーン画像生成 ×4** (各1回)
   - 入力: ベースアバター(B5) + コーデ部位 + ロケーション + ポーズ
   - 出力: シーン画像URL → C14-17

### 横断ルール

- **画像API**: まず **Nano Banana Pro**（Google、Imagen 4系の後継ブランド）で再起動 → 後で GPT Image 2 と比較して本採用を決定（**アダプタ層経由**で差替可能）
- **失敗時挙動**: 前回成功画像を残置（C案）
- **英語化**: シート＆AIプロンプトは全部英語、ユーザー表示の日本語はワンポイントのみ
- **アバター画像再生成**: プロフィール変更時のみ（通常は0回/日）
- **同時操作**: 当面無視（MVP）
